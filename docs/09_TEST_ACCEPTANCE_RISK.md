## 5. 风险清单

### 5.1 技术风险

| 风险 | 影响 | 应对 |
|---|---|---|
| AutoLXB route 无法稳定读取 | 核心闭环受阻 | 第 1 周优先验证 |
| Trace 不含足够上下文 | 失败解释不准 | 产品层补充 RunRecord |
| 单步测试底层不支持 | Route Studio 体验下降 | 用临时 route 替代 |
| 不同 ROM 杀后台 | 定时任务失败 | 权限体检 + 前台服务 |
| 模型视觉能力不稳定 | 首次试跑失败 | Playbook 优化 + 重试 |

### 5.2 产品风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 用户以为任何任务都能做 | 期望过高 | 明确推荐任务类型 |
| Route Studio 太复杂 | 小白不会用 | 默认只展示核心操作 |
| AI 建议误导用户 | 路线被改坏 | 修改前后对比 + 回滚 |
| 高风险误触 | 严重损失 | Safety Guard 强制确认 |

### 5.3 合规风险

| 风险 | 应对 |
|---|---|
| 自动化违反第三方 App 规则 | 用户提示和免责声明 |
| 截图包含隐私并上传模型 | 明确模型配置和隐私提示 |
| 自动发送内容 | 强制确认 |
| 自动支付/转账 | 禁止自动执行 |

## 6. MVP 发布门槛

必须全部满足：

```text
- 10 个内测任务至少 7 个跑通核心流程
- 所有 L2 高风险动作都能被拦截
- L3 禁止动作不会自动执行
- Route Studio P0 操作全部可用
- Core 异常时不会导致 App 崩溃
- Trace 页面能给出基本人话解释
- 用户能从失败记录进入 Route Studio 修复
```
