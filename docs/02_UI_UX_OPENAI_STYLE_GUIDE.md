# 02. UI/UX 设计规范：现代简约、对话式、低噪声

> 项目：SmartTask AI / AI 安卓自动化任务产品  
> 版本：v0.2  
> 日期：2026-05-23  
> 底层参考：AutoLXB 二次开发

## 1. 设计目标

新增开发需求：

> 页面设计采用更现代简约的 UI 风格和操作逻辑，可参考 OpenAI App 的简洁、对话式、低噪声、卡片化、渐进式呈现方式。

注意：这里的“参考 OpenAI App 风格”指的是体验原则，而不是复制品牌标识、图标、商标或视觉资产。

目标体验：

```text
少表单
少术语
少噪声
强对话入口
强状态反馈
强人工可控
```

---

## 2. 设计关键词

| 关键词 | 解释 |
|---|---|
| 现代简约 | 大留白、轻层级、低装饰、清晰文字 |
| 对话式 | 用户先描述目标，系统再逐步补全 |
| 卡片化 | 信息分块呈现，减少复杂表格压迫感 |
| 渐进式 | 默认只展示必要信息，高级选项折叠 |
| 低噪声 | 不堆按钮、不堆配置项、不堆技术日志 |
| 可解释 | AI 每次结论都有“为什么”和“可修改” |
| 可撤销 | 编辑路线时支持撤销、草稿、回滚 |
| 安全感 | 高风险动作醒目、可暂停、可取消 |

---

## 3. 视觉风格方向

### 3.1 总体风格

| 维度 | 规范 |
|---|---|
| 背景 | 浅灰 / 暗色模式下深灰，避免纯白刺眼 |
| 卡片 | 圆角 20-28dp，轻阴影或无阴影边界 |
| 分割线 | 尽量少，用留白和卡片区分层级 |
| 色彩 | 中性色为主，状态色谨慎使用 |
| 字体 | 系统字体，标题清晰，正文可读 |
| 图标 | 线性图标，低装饰性 |
| 动效 | 轻量、短时、表达状态变化 |

### 3.2 色彩建议

不绑定品牌色，建议提供主题 token：

```text
color.bg.primary        # 主背景
color.bg.card           # 卡片背景
color.text.primary      # 主文字
color.text.secondary    # 次级文字
color.text.tertiary     # 弱文字
color.border.subtle     # 轻边框
color.accent            # 主操作色
color.success           # 成功
color.warning           # 风险/注意
color.danger            # 高风险/禁止
```

### 3.3 字体层级

| 场景 | 字号建议 | 权重 |
|---|---:|---|
| 页面标题 | 24-28sp | Semibold |
| 卡片标题 | 18-20sp | Medium/Semibold |
| 正文 | 15-16sp | Regular |
| 辅助说明 | 13-14sp | Regular |
| 标签/状态 | 12-13sp | Medium |

### 3.4 间距

| 场景 | 建议 |
|---|---:|
| 页面左右边距 | 20-24dp |
| 卡片内边距 | 16-20dp |
| 卡片间距 | 12-16dp |
| 按钮高度 | 44-52dp |
| 底部安全区 | 24dp + system inset |

## 4. 操作逻辑原则

### 4.1 从“配置表单”改为“对话输入”

不要让用户先选：

```text
任务类型 / 触发器 / 包名 / playbook / route mode / finish policy
```

而是先问：

```text
你想让手机自动完成什么？
```

用户输入后再展示 AI 解析结果。

### 4.2 高级配置后置

页面默认展示：

- 任务名称。
- 目标 App。
- 触发方式。
- 执行目标。
- 风险等级。
- 开始试跑按钮。

高级配置折叠：

- route mode。
- fallback policy。
- model selection。
- retry count。
- timeout。
- raw playbook。

### 4.3 重要状态要可见

用户始终要知道：

| 状态 | 表达 |
|---|---|
| Core 是否运行 | 首页顶部状态 pill |
| 权限是否正常 | 权限体检卡片 |
| 当前是否调用 AI | 试跑页状态文案 |
| 路线是否在回放 | 执行页步骤高亮 |
| 是否有高风险动作 | 醒目确认面板 |
| 失败位置 | 步骤时间线定位 |

## 5. 信息架构

底部导航建议 4 个一级入口：

```text
首页 / 任务 / 运行记录 / 设置
```

不建议在底部导航放太多入口。

### 5.1 首页

目标：快速创建任务 + 看状态。

```text
首页
├── Core 状态卡
├── 一句话创建输入框
├── 今日执行摘要
├── 需要处理的问题
└── 最近任务
```

### 5.2 任务

目标：管理自动化资产。

```text
任务
├── 全部
├── 手动
├── 定时
├── 通知触发
└── 已暂停
```

### 5.3 运行记录

目标：看成功/失败和修复建议。

```text
运行记录
├── 时间线
├── 成功/失败筛选
├── 模型调用筛选
└── 失败诊断入口
```

### 5.4 设置

目标：配置模型、权限、安全策略。

```text
设置
├── 权限体检
├── 模型配置
├── Core 启动
├── 安全策略
├── 成本预算
└── 导入导出
```

## 6. 核心页面设计

## 6.1 首页

### 页面草图

```text
┌─────────────────────────────┐
│ SmartTask                   │
│ Core 正在运行 · 权限正常       │
├─────────────────────────────┤
│ 你想自动完成什么？             │
│ ┌─────────────────────────┐ │
│ │ 每天早上9点打开淘宝收金币  │ │
│ └─────────────────────────┘ │
│ [创建任务]                  │
├─────────────────────────────┤
│ 今日                         │
│ 成功 8 · 失败 1 · AI 调用 2   │
├─────────────────────────────┤
│ 需要处理                     │
│ 贴吧签到 第3步失败            │
│ [查看原因]                   │
└─────────────────────────────┘
```

### 交互

- 输入框聚焦时底部按钮变为“创建任务”。
- 如果 Core 未运行，创建按钮下方显示“需要先启动 Core”。
- 不在首页展示复杂设置。

---

## 6.2 AI 任务创建页

### 原则

像对话一样推进，不像传统表单。

### 页面结构

```text
用户输入：
“每天早上9点打开淘宝芭芭农场收金币”

AI 解析结果卡片：
- 任务：淘宝芭芭农场收金币
- App：淘宝
- 触发：每天 09:00
- 风险：低
- 执行方式：首次学习，成功后复用路线

[开始试跑]
[调整设置]
```

### 修改方式

点击某一行后弹出轻量底部 sheet：

- 修改任务名。
- 修改 App。
- 修改时间。
- 修改风险策略。

避免把所有字段铺成大表单。

---

## 6.3 首次试跑页

### 目标

让用户感觉“AI 正在学习”，而不是“黑盒卡住”。

### 页面结构

```text
正在学习任务
淘宝芭芭农场收金币

当前步骤：
正在查找“芭芭农场”入口...

[当前屏幕缩略图]

时间线：
✓ 打开淘宝
✓ 进入首页
● 查找入口
○ 点击收取金币
○ 检查结果

[停止]
```

### 状态文案规范

| 状态 | 文案 |
|---|---|
| 调用模型 | 正在理解当前页面 |
| 点击 | 正在点击“我的淘宝” |
| 等待 | 正在等待页面加载 |
| 失败 | 没找到目标控件，准备尝试视觉识别 |
| 成功 | 已完成，并学到一条路线 |

---

## 6.4 路线学习结果页

页面以“我学到了什么”为核心。

```text
已学到路线

1. 打开淘宝
2. 点击“我的淘宝”
3. 点击“芭芭农场”
4. 点击“收取金币”
5. 检查领取结果

后续执行：优先使用路线，必要时才调用 AI
风险等级：低

[进入 Route Studio] [保存并启用]
```

## 6.5 Route Studio

Route Studio 是工具属性最强的页面，但仍需保持简洁。

### 移动端布局建议

移动端不要强行三栏。建议使用“主步骤列表 + 底部详情面板”：

```text
┌─────────────────────────────┐
│ 淘宝收金币 · 路线 v1.0        │
│ [测试] [发布]                │
├─────────────────────────────┤
│ 步骤时间线                   │
│ ✅ 1 打开淘宝                 │
│ ✅ 2 等待首页                 │
│ ⚠️ 3 点击坐标区域             │
│ ✅ 4 点击收取金币             │
├─────────────────────────────┤
│ 选中步骤截图                 │
│ [截图 + 点击点 + 控件框]       │
├─────────────────────────────┤
│ 动作：点击                    │
│ 定位：坐标 x=420 y=810        │
│ AI 建议：改为文本定位          │
│ [编辑] [单步测试]             │
└─────────────────────────────┘
```

平板/横屏可以使用三栏。

### 操作优先级

一级操作：

- 编辑目标。
- 单步测试。
- 禁用步骤。
- 删除步骤。

二级操作：

- 修改重试。
- 修改兜底策略。
- 查看 raw data。
- 锁定步骤。

## 6.6 Trace 解释页

不要直接展示 JSONL。

页面应该先说结论：

```text
任务失败在第 3 步

原因：没有找到“芭芭农场”入口。
可能是 App 首页布局变化，原路线使用坐标点击，位置已经偏移。

建议：
将第 3 步改成文本定位“芭芭农场”。

[进入 Route Studio 修复]
[查看技术日志]
```

技术日志折叠到最下方。

## 7. 组件规范

### 7.1 状态 Pill

用于展示 Core、权限、风险、路线状态。

```text
运行中 / 未启动 / 需要处理 / 高风险 / 草稿 / 已发布
```

### 7.2 任务卡片

内容：

- 任务名。
- 触发方式。
- 最近状态。
- 成功率。
- 快捷操作。

### 7.3 步骤卡片

内容：

- 序号。
- 状态图标。
- 动作摘要。
- 定位方式标签。
- 风险/稳定性提示。

### 7.4 AI 建议卡

必须包括：

- 建议内容。
- 原因。
- 影响。
- 操作按钮：应用 / 忽略 / 查看详情。

AI 不得静默修改。

## 8. 交互动效

| 场景 | 动效建议 |
|---|---|
| 创建任务解析中 | 轻量 loading + 文案变化 |
| 试跑步骤推进 | 时间线 step 逐个点亮 |
| Route Studio 选中步骤 | 截图区域平滑切换 |
| AI 建议出现 | 卡片轻微上浮/淡入 |
| 高风险确认 | 底部 sheet 或全屏 modal，避免误触 |
| 发布成功 | 简短 toast + 状态更新 |

---

## 9. 暗色模式

必须支持暗色模式，因为自动化工具常被夜间使用。

暗色模式原则：

- 不使用纯黑大面积背景，可使用深灰。
- 高风险红色不要过饱和。
- 卡片边界用浅透明描边。
- 截图预览保持真实亮度，不强行调暗。

---

## 10. UI 验收标准

| 项目 | 标准 |
|---|---|
| 首页 | 5 秒内理解“创建任务”和“Core 状态” |
| 任务创建 | 用户无需理解 Tasker 概念即可创建草稿 |
| 首次试跑 | 用户能知道当前在做什么 |
| Route Studio | 用户能找到“编辑、测试、禁用、删除” |
| Trace 解释 | 先给人话结论，再给技术日志 |
| 高风险动作 | 弹窗足够醒目，默认不会误确认 |
| 整体风格 | 简洁、留白、卡片化、低噪声 |
